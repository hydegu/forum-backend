=======================================================
  推送项目到 GitHub - 快速命令清单
=======================================================

第一步：在 GitHub 创建仓库
-----------------------
1. 访问 https://github.com
2. 点击右上角 + → New repository
3. 仓库名称：forum-backend
4. 选择 Private 或 Public
5. ⚠️ 不要勾选任何初始化选项（README、gitignore、license）
6. 点击 Create repository


第二步：复制粘贴以下命令（在当前目录执行）
-----------------------
⚠️ 注意：将 YOUR_USERNAME 替换为你的 GitHub 用户名

# 添加远程仓库
git remote add origin https://github.com/YOUR_USERNAME/forum-backend.git

# 验证远程仓库
git remote -v

# 推送 main 分支
git push -u origin main

# 推送 develop 分支
git push -u origin develop

# 查看所有分支
git branch -a


第三步：验证推送成功
-----------------------
1. 刷新 GitHub 仓库页面
2. 应该能看到：
   - 115 个文件
   - main 和 develop 两个分支
   - 3 次提交


常见问题快速解决
-----------------------

问题 1：Authentication failed（认证失败）
解决：需要使用 Personal Access Token
     GitHub → Settings → Developer settings
     → Personal access tokens → Generate new token
     → 勾选 repo 权限 → 生成 token
     → 推送时用 token 作为密码

问题 2：fatal: repository not found（仓库未找到）
解决：检查仓库地址是否正确
     git remote -v
     git remote remove origin
     git remote add origin https://github.com/用户名/仓库名.git

问题 3：Updates were rejected（更新被拒绝）
解决：git pull origin main --allow-unrelated-histories
     git push origin main


SSH 方式配置（推荐，无需每次输入密码）
-----------------------
# 生成 SSH 密钥
ssh-keygen -t ed25519 -C "degulasihanyu@gmail.com"

# 查看公钥（复制内容）
cat ~/.ssh/id_ed25519.pub

# 添加到 GitHub：
# GitHub → Settings → SSH and GPG keys → New SSH key
# 粘贴公钥，保存

# 修改远程地址为 SSH
git remote set-url origin git@github.com:YOUR_USERNAME/forum-backend.git

# 推送（无需密码）
git push origin main


下一步：日常开发流程
-----------------------

开发新功能：
1. git checkout develop
2. git pull origin develop
3. git checkout -b feature/功能名
4. 编写代码...
5. git add .
6. git commit -m "feat: 功能描述"
7. git push -u origin feature/功能名
8. 在 GitHub 创建 Pull Request

修复 Bug：
1. git checkout develop
2. git checkout -b bugfix/问题描述
3. 修复代码...
4. git commit -m "fix: 问题描述"
5. git push -u origin bugfix/问题描述

微服务改造计划
-----------------------
当前结构：单体应用（Monolith）
    main (生产环境)
    └── develop (开发分支)

未来结构：微服务架构（Microservices）
    forum-user-service      # 用户服务
    forum-post-service      # 帖子服务
    forum-comment-service   # 评论服务
    forum-auth-service      # 认证服务
    forum-gateway           # API 网关

改造步骤：
1. 阶段一：模块化重构（当前仓库）
   git checkout -b refactor/prepare-microservices
   将代码按领域划分模块

2. 阶段二：创建独立仓库
   为每个微服务创建独立的 Git 仓库

3. 阶段三：逐步迁移
   保留单体应用作为参考，新功能在微服务中开发
