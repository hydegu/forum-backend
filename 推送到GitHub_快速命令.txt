常见问题快速解决
-----------------------

问题 1：Authentication failed（认证失败）
解决：需要使用 Personal Access Token
     GitHub → Settings → Developer settings
     → Personal access tokens → Generate new token
     → 勾选 repo 权限 → 生成 token
     → 推送时用 token 作为密码

问题 2：fatal: repository not found（仓库未找到）
解决：检查仓库地址是否正确
     git remote -v
     git remote remove origin
     git remote add origin https://github.com/hydegu/仓库名.git

问题 3：Updates were rejected（更新被拒绝）
解决：git pull origin main --allow-unrelated-histories
     git push origin main


SSH 方式配置（推荐，无需每次输入密码）

# 修改远程地址为 SSH

下一步：日常开发流程
-----------------------

开发新功能：
1. git checkout develop
2. git pull origin develop
3. git checkout -b feature/功能名
4. 编写代码...
5. git add .
6. git commit -m "feat: 功能描述"
7. git push -u origin feature/功能名
8. 在 GitHub 创建 Pull Request

修复 Bug：
1. git checkout develop
2. git checkout -b bugfix/问题描述
3. 修复代码...
4. git commit -m "fix: 问题描述"
5. git push -u origin bugfix/问题描述

微服务改造计划
-----------------------
当前结构：单体应用（Monolith）
    main (生产环境)
    └── develop (开发分支)

未来结构：微服务架构（Microservices）
    forum-parent (Maven 多模块项目)
      ├── forum-common (公共模块)
      ├── forum-user-module (用户模块)
      ├── forum-post-module (帖子模块)
      ├── forum-comment-module (评论模块)
      └── forum-gateway (Spring Cloud Gateway)

改造步骤：
1. 阶段一：创建多模块结构（在当前仓库）
   git checkout -b refactor/maven-multi-module
   创建父 POM 和各子模块骨架
   forum-parent/
     ├── pom.xml (父POM)
     ├── forum-common/
     ├── forum-user-module/
     ├── forum-post-module/
     ├── forum-comment-module/
     └── forum-gateway/

2. 阶段二：领域代码拆分
   将现有代码按业务领域迁移到各子模块：
   - 用户模块：AppUser, 认证相关
   - 帖子模块：Post, 分类相关
   - 评论模块：PostComment 相关
   - 公共模块：共享工具类、配置、DTO

3. 阶段三：模块依赖配置
   配置模块间依赖关系
   调整 application.yml 配置
   统一依赖版本管理

4. 阶段四：测试与验证
   运行测试确保功能正常
   git merge develop
   git push origin refactor/maven-multi-module
   创建 Pull Request 合并到 develop
